What's Inside

Every line of code written from scratch — no Linux kernel, no libc, no shortcuts.

⚙️

Custom Kernel

x86_64 long mode, multiboot2, custom GDT, IDT with full exception handling, PIC remapping, and PIT timer at 1 kHz.

💾

Memory Management

Bitmap physical memory manager supporting up to 256 MB, kernel heap with kmalloc/kfree and block coalescing.

🔐

Login & Auth

First-boot setup wizard (username, timezone, password). Login screen with 10-attempt lockout. Persistent config saved to ATA disk.

🖥️

macOS-Style UI

Status bar with live clock, dark and light themes, CP437 box-drawing UI components, colour-coded shell output.

🌐

Text Browser

Built-in browser with URL bar, scrollable pages, and about: protocol — about:home, about:help, about:system.

💿

ATA Storage

PIO-mode ATA driver for reading and writing 512-byte sectors. Config persists across reboots when a hard disk is attached.

📂

Virtual Filesystem

Read-only ramdisk VFS with ls and cat — /etc, /proc, /boot, and /readme.txt compiled into the kernel.

🔊

PC Speaker

PIT channel 2 speaker driver. Boot chime, configurable beeps. No audio card needed — uses the original IBM PC speaker.

Build & Run

One Docker container, one make command. Works on Windows, macOS, and Linux.

1

Clone the repo

git clone https://github.com/rupertjwakefield2-dot/Torrent-OS.git
cd torrentos
2

Build the Docker environment

docker build -t torrentos-buildenv buildenv/
3

Compile the OS

docker run --rm -it -v $(pwd):/root/env torrentos-buildenv
# inside the container:
make build-x86_64
exit
4

Boot in QEMU

# without persistence
qemu-system-x86_64 \
  -drive file=dist/x86_64/TorrentOS.iso,format=raw,media=cdrom,readonly=on \
  -boot d -m 256M

# with persistent storage
qemu-img create -f raw torrentos.img 64M
qemu-system-x86_64 \
  -drive file=dist/x86_64/TorrentOS.iso,format=raw,media=cdrom,readonly=on \
  -drive file=torrentos.img,format=raw \
  -boot d -m 256M
